# OpenAI Project Candidates

The campaign container is selected: **Public Audition**.

This file tracks possible proof objects that could sit inside the audition.

## Proof Object Options

### 1. Maintainer League

Codex attempts real open-source maintenance tasks in public.

Why it fits:

- Directly tied to Codex.
- Technical receipts are clean.
- Merged PRs are hard to fake.

Risk:

- Less emotionally viral.
- External review cycles may slow the story.

### 2. Public Audition Generator

A lightweight tool that lets another person create their own proof-of-work application plan.

Why it fits:

- Makes the process repeatable.
- Keeps the product about the process.
- Easy to demo.

Risk:

- Can look like a shallow generator unless the Noah/OpenAI case study is strong.

### 3. Codex Work Journal

A public work journal showing what Codex did, what Noah corrected, and what shipped.

Why it fits:

- Transparent.
- Honest.
- Strongly aligned with long-running work and memory.

Risk:

- Needs an interesting output to avoid feeling like meta content only.

### 4. Argus Live

Codex helps build or package an agent investigation system for public adversarial data.

Why it fits:

- Technically strong.
- Public receipts.
- Uses Noah's existing edge.

Risk:

- Crypto narrows the audience.
- It can hijack the narrative.

## Current Recommendation

Launch the public audition first.

Then add one proof object only if it strengthens the core narrative:

> Codex helped Noah get hired by creating public proof, not by writing a cover letter.
