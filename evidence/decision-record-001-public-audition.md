# Decision Record 001: The Project Is The Public Audition

Date: 2026-05-01

Status: Public excerpt

Source: Session 0 planning conversation with Codex.

## Claim

The project shifted from "build a viral app" to "make the public audition process itself the artifact."

## Why It Matters

This is the core move.

If the project were only a generic app, it would need to compete on app novelty. The stronger story is recursive: a human asks Codex to help get hired by OpenAI, and the resulting public process becomes the application.

## Excerpted Prompt

Noah opened with:

> You are me. You want to get hired by OpenAI after your project goes viral. What is your project, what will get the most attention, what could go viral.

The useful correction came later:

> The project needs to go viral somehow, which means it needs to be interesting. The historical artifacts need to be clean and useful once OpenAI people go looking.

## Human Judgment

Noah rejected two weak directions:

- defaulting to an existing crypto/onchain project,
- storing raw chats as if transcript volume were the artifact.

The correction was that each phase needs a different artifact: viral surface, landing page, case file, evidence, and private source material.

## Codex Contribution

Codex translated the correction into a public repo structure:

- `CASE_FILE.md`
- `launch/`
- `evidence/`
- `RECEIPTS.md`
- `BUILD_LOG.md`

Later pruning removed internal planning scaffolding so the public path stayed focused.

## Outcome

The public audition became the selected top-level project.

Existing projects remain candidate proof objects only if they strengthen the central story.
