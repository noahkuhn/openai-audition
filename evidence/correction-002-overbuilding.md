# Evidence 002: The Overbuilding Correction

Date: 2026-05-02

Status: Public excerpt

## Claim

The project became stronger when Noah corrected Codex away from repo polish and back toward the real objective: get noticed by OpenAI.

## What Happened

Codex initially reviewed the repo like a normal launch artifact: broken links, receipt depth, deployment fragility, and static-site details.

Noah pushed back. The correction was that the goal was not a clean repo for its own sake. The goal was to use the repo as an attention object that could start an OpenAI conversation.

## Why It Matters

This is the exact behavior the project is trying to expose:

- Codex can create structure and momentum.
- Codex can also drift into low-leverage polish.
- Human judgment still has to preserve the objective.
- The correction loop is part of the product evidence.

## Resulting Changes

The repo was recentered around:

- a direct Codex role fit,
- an OpenAI-facing case file,
- public launch and outreach copy,
- a public GitHub repo,
- a deployed GitHub Pages site,
- and a stronger first-click homepage.
