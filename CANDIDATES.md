# Candidate Projects

This file exists so the public can see what was considered, what was rejected, and why.

## Candidate 1: Public Audition

One-line hook:

> I asked Codex to get me hired at OpenAI, so it built a public proof-of-work application instead of a resume.

Scores:

- OpenAI narrative fit: 5/5
- Viral clarity: 5/5
- Technical credibility: 3/5
- Ship speed: 5/5
- Proof density: 5/5
- Hiring signal: 4/5
- Repeatability: 5/5

Total: 32/35

Why it might win:

- The meta angle is immediately legible.
- The repo itself is the artifact.
- It aligns with Codex as a long-running work partner.
- It can ship immediately and improve in public.

Why it might fail:

- It can look too performative if there is no real work behind it.
- It needs enough substance to avoid feeling like a content stunt.

First public artifact:

- Static site plus repo with mission, scorecard, build log, prompts, launch thread, and evolving application.

Status:

- Selected as the top-level campaign container.
