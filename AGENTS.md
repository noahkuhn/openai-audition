# Project Instructions

This repo is a public proof-of-work audition.

## Core Rule

The process is the point.

Do not hide planning, rejected ideas, scorecards, prompts, failures, or human corrections. They are the artifact.

## Editing Priorities

1. Preserve the viral premise: "I asked Codex to get me hired at OpenAI."
2. Make the process legible to someone seeing the repo for the first time.
3. Prefer simple public artifacts over deep app complexity.
4. Treat existing Noah projects as candidate material, not default direction.
5. Keep claims honest and verifiable.

## Avoid

- Generic AI startup language.
- Fake autonomy.
- Fake metrics.
- Overbuilding before the public narrative is clear.
- Making the project about crypto, unless that wins through the scorecard.
- Turning the repo into a private notebook.

## Before Finalizing Meaningful Changes

Check:

- Does this strengthen the public audition?
- Does this show useful human-agent collaboration?
- Is there a receipt or build-log entry for meaningful decisions?
- Would someone at OpenAI understand why this matters?
